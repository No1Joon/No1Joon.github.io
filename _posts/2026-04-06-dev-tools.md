## Claude Code

AI 페어 프로그래밍 도구로 코드 생성, 리팩터링, 리뷰에 활용한다.

```bash
# CLI 실행
claude

# 특정 파일 컨텍스트로 대화
claude "이 WebSocket 핸들러에서 메모리 누수 가능성을 찾아줘"
```

주로 활용하는 케이스:
- 반복적인 보일러플레이트 생성 (Pydantic 스키마, SQLAlchemy 모델)
- 비동기 패턴 검토
- 테스트 케이스 생성

## ruff — Python 린트 + 포맷

<div class="callout why">
<div class="callout-title">ruff 하나로 flake8 + isort + black 대체</div>

Rust로 작성되어 기존 Python 도구 대비 **10–100× 빠름**. CI에서 린트 단계 시간이 크게 단축된다.
</div>

```toml
# pyproject.toml
[tool.ruff]
target-version = "py312"
line-length = 100

[tool.ruff.lint]
select = [
    "E",   # pycodestyle errors
    "W",   # pycodestyle warnings
    "F",   # pyflakes
    "I",   # isort
    "B",   # flake8-bugbear
    "UP",  # pyupgrade
]
ignore = ["E501"]  # line length는 formatter에 위임

[tool.ruff.format]
quote-style = "double"
```

```bash
# 린트 + 자동 수정
ruff check . --fix

# 포맷
ruff format .

# CI (수정 없이 검사만)
ruff check . && ruff format --check .
```

## mypy — 정적 타입 체크

```toml
# pyproject.toml
[tool.mypy]
python_version = "3.12"
strict = true
ignore_missing_imports = true

[[tool.mypy.overrides]]
module = ["google.cloud.*", "asyncpg.*"]
ignore_missing_imports = true
```

```bash
# 타입 체크 실행
mypy app/

# 특정 파일
mypy app/services/live_collector.py
```

`strict = true`로 활성화되는 주요 검사:
- `disallow_untyped_defs` — 모든 함수에 타입 어노테이션 강제
- `check_untyped_defs` — 타입 없는 함수 내부도 검사
- `warn_return_any` — `Any` 반환 경고

## ESLint — Frontend 린트

```json
// .eslintrc
{
  "extends": [
    "eslint:recommended",
    "plugin:@typescript-eslint/strict-type-checked",
    "plugin:react-hooks/recommended"
  ],
  "rules": {
    "react-hooks/exhaustive-deps": "error",
    "@typescript-eslint/no-floating-promises": "error"
  }
}
```

`react-hooks/exhaustive-deps`를 `error`로 설정 — 의존성 배열 누락으로 인한 stale closure 버그를 빌드 전에 차단한다.

## 전체 품질 게이트

```bash
# Frontend
pnpm lint        # ESLint
pnpm typecheck   # tsc --noEmit
pnpm test        # Vitest

# Backend
ruff check .
ruff format --check .
mypy app/
pytest
```

CI에서 모든 게이트를 통과해야 머지가 가능하다.
