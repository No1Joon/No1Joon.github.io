## 왜 이 조합인가

<div class="callout why">
<div class="callout-title">React + SVG</div>

Canvas API 대신 **SVG를 선택**해 React state로 그래픽을 직접 제어한다. GSAP의 `ref` 타겟팅이 자연스럽게 연결된다는 점이 결정적이었다.

- Canvas: 픽셀 버퍼 직접 조작 → React 리렌더 사이클과 충돌
- SVG: DOM 노드 → React `ref`로 직접 참조 가능, GSAP 타겟 그대로 사용
</div>

<div class="callout why">
<div class="callout-title">Zustand</div>

Redux 없이 **simulation / live / ui** 세 관심사를 스토어 단위로 분리 관리한다. selector 패턴으로 불필요한 리렌더링을 차단한다.

```ts
// simulation 스토어만 구독 — live 변경 시 리렌더 없음
const packets = useSimulationStore(s => s.packets)
```
</div>

<div class="callout why">
<div class="callout-title">GSAP</div>

`gsap.timeline()`으로 패킷 이동 시퀀스를 선언적으로 표현한다. 재생 / 일시정지 / 배속 제어가 단 한 줄이다.

```ts
const tl = gsap.timeline({ paused: true })
tl.to(packetRef.current, { motionPath: path, duration: 1 })
tl.play()           // 재생
tl.timeScale(2)     // 2배속
tl.pause()          // 일시정지
```
</div>

## TypeScript strict 모드

`tsconfig.json`에 `"strict": true` 설정. 이로 인해 활성화되는 주요 옵션:

- `strictNullChecks` — `undefined` 가능성을 타입 레벨에서 강제
- `noImplicitAny` — 추론 불가 시 에러
- `strictFunctionTypes` — 콜백 타입 공변성 검사
