맥북에서 잘 쓰던 넷플릭스가 어느 날 크롬에서만 "이용 가구가 아닙니다" 오류를 띄우기 시작했습니다 같은 계정·같은 Wi-Fi인데 사파리에선 정상이어서 브라우저 쪽 상태가 엉킨 것으로 보고 그쪽을 건드렸습니다

## 증상

크롬에서 넷플릭스에 로그인하면 재생 전에 "이용 가구가 아닙니다" 안내 화면이 뜨고, 인증 코드 입력 단계로 강제 진입합니다 같은 맥북에서 **사파리로 접속하면 곧바로 정상 재생**됩니다 계정·네트워크·시간대는 모두 동일했습니다

## 환경

| 항목       | 값                     |
| ---------- | ---------------------- |
| 디바이스   | MacBook (M3)           |
| OS         | macOS                  |
| 브라우저 A | Chrome — 오류 발생     |
| 브라우저 B | Safari — 정상          |
| 계정       | 개인 Netflix 프로필    |
| 네트워크   | 집 Wi-Fi (동일 공유기) |

당시 크롬·macOS 상세 버전을 기록해두지 않아서, 이 글의 재현성은 "맥북 + 크롬" 조합 수준까지만 보장합니다

## 시도한 것

각 시도의 결과를 함께 적습니다 다른 블로그나 AI 답변에서 자주 언급되는 항목도 "미시도" 로 같이 남겼습니다

| 시도                                  | 결과          |
| ------------------------------------- | ------------- |
| 사파리에서 재접속                     | 정상 (비교용) |
| 크롬에서 넷플릭스 관련 쿠키·캐시 삭제 | **해결**      |
| 확장 프로그램 비활성화                | 미시도        |
| 크롬 재시작 / 시크릿 창               | 미시도        |
| 넷플릭스 "이용 가구 등록" 인증 코드   | 미시도        |

쿠키 삭제 하나로 끝났기 때문에 나머지 후보는 검증해보지 못했습니다

## 해결

크롬에서 넷플릭스 관련 저장 데이터를 **전부** 지운 뒤 재로그인하니 정상으로 돌아왔습니다

1. 크롬 주소창에 `chrome://settings/content/all` 입력
2. 상단 우측 `Filter sites on page` 검색창에 `netflix` 입력 → netflix.com 관련 항목 필터
3. 해당 도메인의 데이터 **전체 삭제** (쿠키·사이트 데이터·캐시 포함)
4. 넷플릭스 재접속 → 재로그인 → 정상 재생 확인

![chrome://settings/content/all 화면에서 상단 Filter sites on page 검색창에 netflix 를 입력해 관련 도메인만 필터링한 모습](/assets/images/daily-dev/2026-04-22-netflix-household-chrome/chrome-settings-filter.webp)

핵심은 "netflix.com 저장 데이터를 한 번에 싹 지우는 것" 이었습니다 일반 캐시만 비우는 `Cmd + Shift + Delete` 경로로도 같은 결과가 나오는지는 확인하지 않았습니다

## 원인 추정

원인은 **불명**입니다

크롬에 남아있던 이전 세션의 넷플릭스 쿠키가 "다른 가구" 로 판단되도록 신호를 보내고 있었던 것으로 추측은 되지만, 넷플릭스가 가구 판별에 내부적으로 어떤 필드를 쓰는지는 공개되어 있지 않아 추측 이상으로 말할 근거가 없습니다 AI 답변에서 자주 등장하는 "HDR·DRM 모듈 충돌이 가구 오류로 오인된다" 는 설명 역시, 쿠키 삭제 한 단계로 해결된 이번 사례에는 맞지 않아 보입니다

## 아직 모르는 것

- **왜 크롬만** 그랬는가 — 사파리는 같은 기기·같은 네트워크인데도 멀쩡했음
- **재발 여부** — 며칠 뒤 다시 발생할지, 어떤 주기로 브라우저별 상태가 엉키는지
- **최소 삭제 단위** — netflix.com 전체가 아니라 특정 쿠키 하나만 지워도 되는지

재발하거나 추가 정보가 생기면 이 글을 업데이트할 예정입니다
