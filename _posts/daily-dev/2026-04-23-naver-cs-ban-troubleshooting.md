개발 및 블로그 운영 계정을 분리하기 위해 네이버 회원가입을 시도하던 중, 아무런 부정 행위 없이 가입 즉시 영구정지를 당하는 당혹스러운 경험을 했습니다. 네이버의 자동화된 고객 응대 시스템과 어뷰징 탐지 로직의 한계를 기록으로 남깁니다

## 증상

업무용 번호를 사용하여 정상적인 가입 절차를 밟았으나, 가입 완료와 동시에 **"대량 생성 ID"**로 판단되어 로그인이 제한되었습니다. 별도의 매크로나 봇을 사용하지 않았음에도 불구하고 소명 기회 없이 즉시 영구정지 처리되었습니다

## 환경

| 항목      | 내용                                      |
| --------- | ----------------------------------------- |
| 기기      | macOS (Claude Code 동시 실행 중)          |
| 네트워크  | 가정용 개인 네트워크                      |
| 브라우저  | Chrome (신규 구글 계정으로 로그인된 세션) |
| 인증 번호 | 업무용 번호 (가입 제한 규정 준수 상태)    |

## 시도한 것

| 시도                                        | 결과                                               |
| ------------------------------------------- | -------------------------------------------------- |
| 1차 고객센터 문의 (정황 설명 및 근거 요구)  | **실패** (AI 자동 응답 기반의 규정 복붙 답변 수신) |
| 2차 고객센터 문의 (아이디 네이밍 가설 제시) | 답변 대기 중                                       |

## 해결 못함..

현재까지도 해결은 못했으며... 네이버 고객센터는 유선 상담 창구가 없으며, 1차 문의 시 키워드 매칭을 통한 **AI 템플릿 답변**이 기본적으로 돌아옵니다...

## 원인 추정

가장 유력한 원인은 **아이디 네이밍 패턴과 신규 환경의 조합**입니다.

- **아이디 패턴**: `jjrottensweetpotato`라는 긴 영문 조합이 봇의 랜덤 생성 알고리즘과 유사하게 판정되었을 가능성이 높습니다
- **신뢰 점수**: 신규 구글 계정과 새로운 브라우저 환경에서 시도된 가입이 어뷰징 필터의 휴리스틱 점수를 높인 것으로 보입니다

## 아직 모르는 것

- **최종 해제 여부**: 현재 재문의 답변을 기다리고 있습니다
- **최소 탐지 기준**: 네이버가 가구 판별이나 어뷰징 탐지에 정확히 어떤 필드를 가중치로 쓰는지 알 수 없습니다

네이버 고객센터의 구조적인 문제와 AI 응답 시스템은 사용자에게 정말 너무나도 큰 피로감을 줍니다. 결과가 업데이트되는 대로 내용을 보충하겠습니다
